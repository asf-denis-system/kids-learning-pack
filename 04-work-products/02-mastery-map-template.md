## 🛡️ GUARD POLICY: Когда переходим на следующий Z-принцип?

**Явный comparator** — правило принятия решения по итогам месяца.

### Критерий «Z-принцип усвоен» (все три обязательны)

| # | Условие | Выполнено? |
|---|---------|-----------|
| 1 | **Частота:** Level 3 зафиксирован ≥ 3 раза за последние 2 недели | ☐ |
| 2 | **Разнообразие:** перенос в ≥ 2 разных контекстах | ☐ |
| 3 | **Спонтанность:** все наблюдения — без вопроса взрослого | ☐ |

→ Все три ☑ → **Z-принцип усвоен**, переходим / добавляем следующий.

### Критерий «Оставаться ещё месяц» (достаточно одного)

- Level 3 наблюдался < 3 раз
- Только 1 контекст (нет переноса в другое место)
- Ребёнок применяет только после вопроса взрослого (Level 2, не 3)
- Красные флаги: отказ или стресс ≥ 2 раза за неделю

→ **Продолжаем текущий Z-принцип**: расширяем контексты или снижаем сложность.

### Критерий «Добавить второй Z-принцип параллельно»

Все три критерия «усвоен» выполнены **+** ребёнок сам задаёт вопросы, скучает, требует «ещё»:

→ **Добавляем второй Z-принцип** (но не более двух одновременно).

### Алгоритм решения

```
Конец месяца → смотрю лог недель

Level 3 × ≥3 раз / в ≥2 контекстах / без подсказки?
  ДА → усвоен ✓
       → ребёнок просит ещё? → добавить второй Z-принцип параллельно
       → нет → переходим на следующий по очереди (см. 03-methods/00-index.md)
  НЕТ → что мешает?
       → мало контекстов → расширить места эпизодов
       → применяет только с подсказкой → убрать подсказку, добавить паузу
       → красные флаги → снизить нагрузку, поднять интерес

Красные флаги ≥ 2/неделю?
  ДА → стоп, пауза 1 неделя, потом переоценка
  НЕТ → продолжаем штатно
```

**Связь со шкалой 0–3:** Level 3 (⭐) = «переносит в новую ситуацию без подсказки» — это и есть единица измерения готовности. Целевая доля эпизодов на Level 3: **60–80%** за последние 2 недели месяца.

# Evidence Register: Проверяемость количественных утверждений

**Назначение:** Зафиксировать, откуда взялись числовые claims в pack, на каких данных они основаны и как пересчитываются.
**Читай это, если:** Нужна трассировка "утверждение -> данные -> метод расчёта -> решение".

---

## Контур применимости

- **Срез:** MVP-пилоты семей с детьми 3-10 лет
- **Период:** февраль 2026 (обновляется при новых волнах)
- **Минимальные источники данных:** еженедельные логи, monthly mastery map, рефлексия ведущих наставников

---

## Реестр claims

| Claim ID | Утверждение | Где используется | Набор данных | Метод расчёта | F-G-R (черновой) | Статус |
|---|---|---|---|---|---|---|
| CLM-001 | "60-80% эпизодов дают спонтанность" | README, Mastery Map Guard Policy | Логи эпизодов + Mastery Map за последние 2 недели цикла | Доля наблюдений уровня 3 (без подсказки) от всех эпизодов в окне | F2 / G: MVP-семьи 3-10 / R: preliminary | Требует регулярного пересчёта |
| CLM-002 | "0-10% после 4 недель = сигнал на пересмотр" | README | Логи эпизодов, срез по принципу за 4 недели | Доля спонтанных переносов (L3) в конце цикла | F2 / G: MVP-семьи 3-10 / R: preliminary | Порог гипотезы, не клиническая норма |
| CLM-003 | "60-70% детей начинают применять спонтанно за месяц" | 06-sota | Агрегированные карты мастерства пилотов + weekly logs | Доля детей с >= 1 L3-переносом в месяце | F1 / G: пилотные семьи / R: preliminary | Уточнить n и окно |
| CLM-004 | "5% -> 40-50% -> 70-80%" (динамика) | 06-sota | Последовательные замеры по неделям/месяцам | Доли по этапам: старт / месяц / квартал | F1 / G: пилотные семьи / R: exploratory | Нужна валидация на расширенной выборке |

## Правило обновления реестра

1. Любое новое числовое утверждение получает `Claim ID`.
2. Без источника данных и метода расчёта claim не поднимается в `README`.
3. При изменении формулы или окна наблюдения обновляется строка claim и дата ревизии.
4. Для публичных материалов публикуется только claim с явным статусом надёжности.

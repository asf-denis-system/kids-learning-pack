## 📋 О Pack'е

Это формализованное инженерное описание **области "обучение детей через нулевые и первые принципы"** — не популярный курс, а систематизированная область инженерных знаний, пригодная для:

- Построения **методик родительского наставничества**
- Разработки **ИИ-ассистентов для детского обучения**
- **Аудита практик** в детских садах и начальной школе
- Построения **типовых траекторий развития** мастерства

---

## 🎯 Целевой срез (TargetSlice)

| Параметр | Значение |
|----------|---------|
| **Возраст** | 3–10 лет: дошкольный трек (3–6) и школьный трек (7–10) |
| **Контексты** | Дом, детский сад, школа, прогулки |
| **Горизонт** | Непрерывный: ~1 месяц на принцип, 8–16 месяцев на все Z0–Z7 |
| **Целевая система** | Ребёнок как агент, носитель изменяемого состояния мастерства объяснять и рассуждать |
| **Ограничения** | Без клинической диагностики; при выраженных признаках тревожности или систематическом стрессе — консультация специалиста |

## 📈 Модель прогрессии (по возрасту и глубине)

Программа строится как **непрерывная**, без фиксированного срока. Ребёнок проходит принципы Z0–Z7 последовательно, меняя глубину по мере взросления.

| Глубина | Что умеет | Возраст (ориентир) | Трек |
|---------|-----------|-------------------|------|
| **@1 — Узнаёт** | Называет принцип в знакомых ситуациях, различает «с подсказкой» | 3–6 лет | Дошкольный |
| **@2 — Объясняет** | Формулирует своими словами, переносит в 2–3 контекста без подсказки | 7–10 лет | Школьный |
| **@3 — Применяет** | Использует в новых задачах, проверяет гипотезы | 11+ лет | Вне MVP |

Ключевой принцип: **один Z-принцип — около месяца практики**. Прогресс измеряется через Guard Policy (см. [04-work-products/02-mastery-map-template.md](04-work-products/02-mastery-map-template.md)).

## 🔧 Ключевые работающие объекты области

### 1. **Нулевые принципы ZP-METHOD v2.0** (Z0–Z7)

| Z | Принцип | Детское различение |
|---|---------|-------------------|
| Z0 | Инварианты | Изменилось ↔ Осталось тем же |
| Z1 | Симметрии операций | Порядок важен ↔ Порядок не важен |
| Z2 | Композиция / системность | Часть ↔ Целое |
| Z3 | Оптимизация | Предмет ↔ Функция |
| Z4 | Неопределённость / вероятность | Точно ↔ Скорее всего ↔ Неизвестно |
| Z5 | Многомасштабность | Вблизи ↔ Издалека |
| Z6 | Ресурсные ограничения | Модель ↔ Реальность |
| Z7 | Интервенция / причинность | От чего это зависит |

Подробно: [01-domain-contract/01B-distinctions.md](01-domain-contract/01B-distinctions.md)
Карточки эпизодов: [03-methods/00-index.md](03-methods/00-index.md)

### 2. **Первые принципы** (объяснения и причинность)
- Причина vs Следствие
- Версия/Гипотеза → Проверка → Уточнение
- Система и её границы
- Контрпримеры и исключения

### 3. **Характеристики мастерства** (что "растим"?)
- Агентность (способность инициировать)
- Саморегуляция (способность удерживать рамку)
- Перенос (способность применить в новом контексте)
- Устойчивость (способность работать при помехах)
- Удержание цели (способность не забыть замысел)
- Качество объяснения (способность аргументировать)
- Работа с ошибкой (способность не сдаться)

## ⚙️ Механизм изменения состояния мастерства

| Условие | Описание |
|---------|---------|
| **Повторяемость** | Не разовый эпизод, а серия эпизодов |
| **Вариативность** | Изменяемость контекста для переноса навыка |
| **Обратная связь** | Сигнал о соответствии действия критерию |
| **Допустимая фрустрация** | Сложность немного выше текущего состояния (ZPD) |
| **Плотность** | Интервалы между эпизодами не разрушают когнитивную связность |

**Отсутствие любого фактора снижает вероятность перехода состояния.**

## 👥 Главные роли

| Роль | Ответственность |
|------|-----------------|
| **Ребёнок (агент обучения)** | Участвовать в эпизодах, проявлять текущее состояние, получать обратную связь |
| **Взрослый-наставник (система создания мастерства)** | Выбрать цель, проектировать среду/ритм, вести эпизоды, наблюдать, корректировать |
| **Со-наставники** | Подхватывать отдельные эпизоды, поддерживать единый стиль |

## 📊 Главные различения (Distinctions)

1. **Навык vs Мастерство мышления**  
   — Навык = умение выполнить процедуру; Мастерство мышления = способность разбираться с новым

2. **Знание vs Умение объяснять**  
   — Знание = факт в голове; Умение = способность применить и аргументировать

3. **Карго-культ vs Реальное развитие**  
   — Карго = внешние атрибуты (много занятий, красивые материалы); Реальное = изменение поведения в быту

4. **Форма обучения vs Содержание обучения**  
   — Форма = как проводить (игра, беседа, эпизод); Содержание = что именно развиваем

5. **Фундаментальное обучение vs Прикладное обучение**  
   — Фундаментальное = практики интеллект-стека (мышление); Прикладное = конкретные знания по предметам

## 📈 Индикаторы прогресса (наблюдаемые проявления)

Ребёнок через 2–4 недели начинает:
- ✓ Говорить «осталось то же самое» при изменении формы **(Z0)**
- ✓ Предсказывать: «если поменять порядок — будет иначе» **(Z1)**
- ✓ **Разложить целое на части** и связать свойства **(Z2)**
- ✓ Называть функцию предмета и **критерий «лучше»** **(Z3)**
- ✓ Говорить «скорее всего» вместо «точно» **(Z4)**
- ✓ Видеть один объект на **разных уровнях** **(Z5)**
- ✓ Отличать **модель от реальности** **(Z6)**
- ✓ Менять **одно** и наблюдать что изменилось **(Z7)**
- ✓ **Реже ждать** подсказку взрослого

Взрослый-наставник через 2–4 недели:
- ✓ Овладел различениями нулевых/первых принципов
- ✓ Может запустить **короткий эпизод-исследование** (5–10 мин)
- ✓ Ведёт **еженедельный лог** наблюдений
- ✓ Способен **регулировать сложность** на лету
- ✓ Понимает, как отличить **реальный перенос от случайности**

## 🔬 SoTA (State of the Art) в области

| Направление | Текущий уровень | Проблема |
|-------------|-----------------|-----------|
| Массовое дошкольное образование | Навыкообучение (чтение, счёт, дисциплина) | Нет явной работы с «интеллект-стеком» |
| Частные развивашки | Игровые формы + навыки | Часто карго-культ: много активности, мало переноса |
| Исследовательское развитие (МИМ) | Систематизация нулевых/первых принципов | Протестировано в пилотах, нужна масштабизация |
| ИИ-ассистенты | Генеративные модели, но без методологии обучения | Нужна структурированная онтология |

## 🛡️ Защита от системных ошибок

| Ошибка | Защита |
|--------|--------|
| Подмена цели (результаты вместо мастерства) | Метрики через поведение в быту, а не через оценки |
| Карго-культ (форма без причины) | Обязательна обратная связь: есть перенос? |
| Перегруз ребёнка | Синтаксис: если состояние ухудшается — снизить нагрузку |
| Рассинхрон взрослых | Явный контракт между взрослыми в начале |
| Ломка агентности | Запрет на стыд, давление, контроль; приоритет — состояние ребёнка |

## 🚀 Как использовать этот pack

**Для родителей/наставников:**
1. Прочитать мин-проверку (section 1.3–1.5)
2. Выбрать 1 Z-принцип для старта (рекомендуется: Z2 или Z0) → [03-methods/00-index.md](03-methods/00-index.md)
3. Найти карточку-эпизод в 03-methods (подходящий принцип и контекст)
4. Провести 1–2 пробных эпизода
5. Вести еженедельный лог (3–4 эпизода/неделю)
6. Каждый месяц обновлять карту мастерства

**Для методистов/разработчиков ИИ:**
1. Понять онтологию (01-domain-contract + 02-domain-entities)
2. Изучить mechanisms (03-methods)
3. Использовать failure-modes как дизайн-юм (чего не делать)
4. Встроить SoTA (06-sota) как контекст для ИИ-подсказок
